GPS411 — IoT Motorcycle Anti-Theft & Real-Time Cellular Tracking System

Powered by STMicroelectronics STM32F411CEU6 • SIMCom A7680C 4G LTE • u-blox NEO-7M GPS • SW-420 Vibration Sensor
01 Executive Summary & Keywords

An advanced, end-to-end IoT motorcycle security system combining real-time positioning and a deterministic multi-stage escalation state machine using an STM32F411 microcontroller. The system parses high-precision coordinates and velocity from a u-blox NEO-7M GPS receiver and senses mechanical disturbances via an SW-420 vibration sensor. Cellular telemetry is executed by a SIMCom A7680C 4G LTE module, transmitting JSON payloads to a cloud-hosted Node.js/Render REST API utilizing optimized HTTP Session Reuse. A responsive Leaflet.js web dashboard delivers live tracking with auto-follow, remote arming/disarming, and instant emergency horn overrides.

Keywords: IoT; STM32F411; A7680C 4G LTE; NEO-7M GPS; SW-420 Sensor; Real-Time Tracking; Multi-Tier Anti-Theft; HTTP Session Reuse; REST API; Leaflet.js.
02 Problem Statement & Solution

Motorcycle theft is a persistent challenge. Traditional acoustic alarms only act as local deterrents within earshot, while standalone GPS trackers lack immediate physical deterrents and often suffer from high cellular latency or heavy data costs. GPS411 resolves this duality by implementing a split-edge/cloud architecture:

  • Deterministic Edge Defense: A multi-stage acoustic escalation system on the STM32F411 processes shocks locally in milliseconds, warning accidental passersby (`ALERT_L1`) while unleashing a full siren (`ALERT_L2`) against persistent theft attempts.
  • Low-Latency Cellular Telemetry: Persistent HTTP session connections reduce 4G transmission latency by over 60%, guaranteeing instant nationwide remote control and real-time tracking on smartphones and computers.
03 System Architecture & High-Tech Block Diagram

The system is built around the 32-bit ARM Cortex-M4 STM32F411CEU6 running a non-blocking super-loop architecture. Peripheral interfacing is segmented across dedicated hardware buses: USART1 (9600 baud) parses NMEA-0183 sentences from the NEO-7M GPS, USART2 (115200 baud) manages AT commands with the A7680C 4G LTE module, and digital GPIO lines process shock interrupts and drive the active acoustic buzzer.

05 Firmware State Machine & Escalation Logic

The firmware eliminates RTOS overhead by deploying a deterministic, non-blocking super-loop (`while(1)`) governed by HAL_GetTick() timers: 

       [TRACKING] <--(Overspeed >60km/h)--> [Beep Warning]
           |
      (Lock / Arm)
           v
       [STANDBY]  <--(Shock detected)-----> [ALERT_L1 (Beep)]
                                                 |
                                     (Repeated shocks inside window)
                                                 v
                                          [ALERT_L2 (Siren)]
  • TRACKING: Active while driving. Constantly monitors speed_kmh from NEO-7M; emits warning beeps if exceeding `60 km/h`.
  • STANDBY: Vehicle locked (`is_armed = 1`). Listens to `PA0` for initial vibrations.
  • ALERT_L1 (Warning): Intermittent beep pattern (200ms ON / 800ms OFF) for accidental touches. Counts cumulative shocks within `L1_TO_L2_WINDOW_MS`.
  • ALERT_L2 (Full Siren): Triggered when `vib_count ≥ L1_TO_L2_COUNT`. Continuous high-decibel siren for 60s or until remote disarm.
  • HORN OVERRIDE: Immediate remote cutoff (`Alert_HornOff()`) overrides all active alarm states instantly.
06 HTTP Session Reuse & Cellular Telemetry

Standard cellular HTTP POST cycles require ~20 seconds due to repeated SSL/TLS handshakes (`AT+HTTPINIT`, `CSSLCFG`, `SSLCFG`). GPS411 implements Persistent HTTP Session Reuse, maintaining the SSL socket open between transmission intervals:

  • First Cycle (`POST`): Executes full 14-step setup (`STOP` → `SSL` → `INIT` → `POST` → `DONE`).
  • Reused Cycles (`REUSE`): Skips directly to step 9 (`DATA` → `BODY` → `ACTION` → `READ` → `DONE`).
  • Performance Gain: Cycle latency reduced from ~20s down to ~3–5s. If network dropouts occur, auto-recovery logic (`*ATREADY:`) seamlessly re-initializes the session.

07 Cloud Server & Leaflet.js Web Dashboard

The cloud backend runs on Node.js / Express.js deployed on Render Cloud, buffering the last 50 GPS waypoints (`history`) and queuing asynchronous control commands:

  • Smart Auto-Follow (`panTo`): Leaflet.js automatically pans to follow the vehicle smoothly. When the user manually drags the map to inspect streets, auto-follow yields control instantly, showing a "📍 Center" snap-back button.
  • Real-Time 2-Second Polling (`2000 ms`): Live status indicators for GPS Fix (`#68d391`), vehicle velocity, protection status, and horn state (`📢 Đang kêu`).
  • Low Cellular Footprint: Highly optimized payloads consume only ~10 MB – 40 MB per day, allowing continuous 24/7 telemetry on basic SIM plans.

08 Experimental Verification, Key Results & Future Work

🧪 Verified Performance & Reliability

  • End-to-End Latency: Verified cellular round-trip time of ~3 to 5 seconds using HTTP Session Reuse, compared to >19s in unoptimized polling mode.
  • Acoustic Override Response: Web-initiated `HORN_OFF` successfully overrides active `ALERT_L1/L2` sirens nationwide in $< 4\text{ seconds}$, preventing false alarm annoyances.
  • Shock Filtering: The two-tier `vib_count` window accurately filters out single accidental bumps (`ALERT_L1`) while escalating reliably against persistent physical manipulation (`ALERT_L2`).

🚀 Conclusion & Future Roadmap

The GPS411 architecture validates that splitting edge deterministic control (STM32F411) from cellular networking (A7680C) creates a highly resilient, low-latency, and cost-effective motorcycle security system.

  • Protocol Migration: Transitioning from HTTP POST polling to MQTT over SSL / WebSockets to achieve sub-second bidirectional command latency.
  • Geofencing & SMS Fallback: Implementing virtual geofence radii that trigger direct cellular SMS alerts when leaving designated boundaries.
  • ADC Battery Monitoring: Sampling vehicle battery voltage (`PB0/ADC1`) to alert users before battery drain occurs.
