GPS411 — IoT Motorcycle Anti-Theft & Real-Time Tracking System

Designed with STM32F411CEU6 • SIMCom A7680C 4G LTE • u-blox NEO-6M GPS • SW-420 Sensor

Abstract

An end-to-end IoT motorcycle security system combines real-time GPS tracking and multi-stage vibration alarm logic using an STM32F411 microcontroller. The system reads precise coordinates and velocity from a NEO-6M GPS sensor and detects mechanical disturbances using an SW-420 vibration sensor. Cellular connectivity is handled by a SIMCom A7680C 4G LTE module, transmitting JSON payloads to a cloud-hosted Node.js/Render REST API via persistent HTTP POST sessions. A responsive Leaflet.js web dashboard provides live map tracking with auto-follow, remote arming/disarming, and instant horn override capabilities.

Keywords: IoT; STM32F411; 4G LTE A7680C; NEO-6M GPS; SW-420 Vibration Sensor; Real-Time Tracking; Anti-Theft State Machine; REST API; Leaflet.js.

I. Introduction

Motorcycle theft remains a critical issue in urban areas. Traditional acoustic alarms only deter thieves locally within earshot, while standalone GPS trackers lack immediate physical deterrents. GPS411 bridges this gap by splitting the architecture into deterministic edge control and cloud-based monitoring:

  • Local Deterrent: A multi-stage acoustic escalation system on the STM32F411 immediately responds to unauthorized vibrations.
  • Remote Awareness: Live telemetry and bidirectional control over nationwide 4G LTE cellular networks, ensuring high availability regardless of distance.

II. System Architecture

The STM32F411 acts as the central processing unit, communicating with peripherals via dedicated interfaces:

  • NEO-6M GPS Module (USART1, 9600 baud) continuously feeds NMEA sentences for position and velocity extraction.
  • SW-420 Vibration Sensor (GPIO PA0) triggers digital pulses upon physical shock.
  • Buzzer (GPIO PA1) executes non-blocking acoustic patterns.
  • A7680C 4G LTE Module (USART2, 115200 baud) bridges the MCU to the cloud server using AT commands over HTTPS/HTTP.

IV. Firmware — STM32 State Machine

The firmware operates on a non-blocking super-loop (`while(1)`) utilizing HAL_GetTick() to ensure simultaneous sensor parsing, UART telemetry, and acoustic escalation without deadlocks.

[TRACKING] <---(Overspeed > 60km/h)---> [Beep Warning] | (Lock / Arm command) v [STANDBY] <---(Vibration detected)---> [ALERT_L1 (Warning Beep)] | (Repeated shocks in window) v [ALERT_L2 (Siren)]
  • TRACKING State: Active while driving. Checks speed_kmh; triggers a short beep warning if speed exceeds 60 km/h.
  • STANDBY State: System armed (is_armed = 1). Monitors PA0 for mechanical shock.
  • ALERT_L1 (Warning): Intermittent beeping (200ms ON / 800ms OFF) for brief disturbances. Counts shocks within a time window (L1_TO_L2_WINDOW_MS).
  • ALERT_L2 (Full Alarm): Continuous siren if disturbances exceed L1_TO_L2_COUNT. Auto-resets after 60s or upon remote disarm.
  • HORN OVERRIDE: Remote HORN_ON/HORN_OFF commands bypass all states instantly, giving the user absolute control over the acoustic alarm.

V. Communication & Session Reuse

To overcome standard 4G HTTP connection latency (~20s per cycle due to repeated SSL handshakes), the firmware implements HTTP Session Reuse:

  • Initial Boot: Full setup (AT+HTTPINIT, SSL config, APN setup).
  • Optimized Loop: Only executes HTTPDATA → Payload → HTTPACTION=1 → HTTPREAD.

  • Smart Auto-Follow (`panTo`): Smoothly centers the map on the vehicle. Automatically yields control when dragging, with a "📍 Center" snap-back button.

VII. Results & Conclusion

  • Optimized Latency: HTTP Session Reuse reduced 4G transmission cycle time from ~20s down to ~3–5s, enabling near real-time tracking.
  • Data Efficiency: Lightweight JSON payloads consume only ~10 MB – 40 MB per day, suitable for standard IoT SIM plans.
  • Reliable Deterrent: The 2-tier alarm successfully filters out accidental bumps while reliably triggering full sirens during theft attempts. Emergency cut-off (`HORN_OFF`) silences alarms nationwide in under 4 seconds.

Future Work: Transition from HTTP to MQTT over SSL / WebSockets for sub-second bidirectional control, implement Geofencing alerts, and add Battery Voltage ADC monitoring.
